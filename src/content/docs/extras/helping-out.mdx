## Writing Documentation

Amina is a complex bot with many features, and comprehensive documentation is crucial for users to fully understand and utilize its capabilities.

<Steps>
1. Familiarize yourself with Amina's features and current documentation.
2. Identify areas where documentation can be improved or expanded.
3. Contact us on our [support server](https://discord.gg/uMgS9evnmv) or create an issue on [GitHub](https://github.com/vixshan/amina) to discuss your ideas.
4. Once approved, start working on the documentation improvements.
5. Submit your changes via a pull request on GitHub.
</Steps>

## Writing Code

Amina is open-source and written in discord.js. You can view and contribute to its code on [GitHub](https://github.com/vixshan/amina).

<Aside type="caution">
  Please only add features if you are proficient in JavaScript. If you're new to GitHub, you can learn how to make changes by watching [this tutorial](https://www.youtube.com/watch?v=yr6IzOGoMsQ).
</Aside>

<Steps>
1. Fork the Amina repository on GitHub.
2. Clone your fork to your local machine.
3. Create a new branch for your feature or bug fix.
4. Make your changes and commit them with clear, concise commit messages.
5. Push your changes to your fork on GitHub.
6. Open a pull request to the main Amina repository.
</Steps>

## Helping in the Support Server

If you're knowledgeable about Amina and want to help others, join our [support server](https://discord.gg/uMgS9evnmv) and start assisting other users.

### Requirements for Support Staff

- Active on the server for at least 2 weeks
- Show initiative in troubleshooting problems and handling general queries
- Able to deal with circumstances independently and trust your own judgment
